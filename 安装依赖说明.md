# 安装依赖说明

## 网络问题解决方案

如果遇到网络超时，可以使用以下方法：

### 方法1：使用国内镜像源（推荐）

```bash
cd GrabOrderApp

# 使用脚本（自动使用清华镜像）
./install_deps_cn.sh

# 或手动指定镜像源
pip3 install -i https://pypi.tuna.tsinghua.edu.cn/simple kivy pillow requests
```

### 方法2：使用其他镜像源

```bash
# 阿里云镜像
pip3 install -i https://mirrors.aliyun.com/pypi/simple/ kivy pillow requests

# 腾讯云镜像
pip3 install -i https://mirrors.cloud.tencent.com/pypi/simple/ kivy pillow requests

# 豆瓣镜像
pip3 install -i https://pypi.douban.com/simple/ kivy pillow requests
```

### 方法3：永久配置镜像源

创建或编辑 `~/.pip/pip.conf`（Mac/Linux）：

```ini
[global]
index-url = https://pypi.tuna.tsinghua.edu.cn/simple
[install]
trusted-host = pypi.tuna.tsinghua.edu.cn
```

然后直接安装：
```bash
pip3 install kivy pillow requests
```

## 跳过PC预览

**PC预览不是必需的！**

如果安装依赖遇到问题，可以：
1. ✅ 直接提交代码到GitHub
2. ✅ 触发GitHub Actions构建APK
3. ✅ 在手机上测试APK（更接近真实环境）

PC预览主要用于：
- 快速验证UI布局
- 检查代码是否有明显错误
- 测试基本功能

但即使跳过PC预览，APK构建后也能在手机上完整测试。

## 推荐流程

### 选项A：快速构建（跳过预览）

```bash
cd GrabOrderApp

# 提交代码
git add -A
git commit -m "修复日志输出，准备构建APK"
git push

# 然后去GitHub Actions查看构建进度
```

### 选项B：先预览再构建

```bash
cd GrabOrderApp

# 1. 安装依赖（使用镜像源）
./install_deps_cn.sh

# 2. 预览
python3 main.py

# 3. 确认无误后提交构建
git add -A
git commit -m "预览测试通过"
git push
```

## 总结

- **网络慢？** → 使用镜像源或跳过预览
- **想快速测试？** → 直接构建APK，在手机上测试
- **想验证代码？** → 使用镜像源安装依赖并预览
