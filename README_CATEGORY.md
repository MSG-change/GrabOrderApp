# 产品分类ID说明

## 已知的产品分类

| 分类ID | 分类名称 | 说明 |
|--------|----------|------|
| 131 | 考核单 | 考核单订单分类 |
| 2469 | 默认分类 | 原默认商品分类 |

## 问题修复

### 原问题
- APP默认使用分类ID `2469`
- 但考核单使用的是分类ID `131`
- 导致查询不到订单

### 解决方案
1. 默认分类ID已改为 `131`（考核单）
2. 可以在UI界面手动修改分类ID
3. 支持动态切换分类

## 使用方法

### 方法1：使用默认值
- 默认已设置为 `131`（考核单）
- 直接启动即可

### 方法2：手动修改
- 在界面的"商品类别"输入框中
- 输入需要的分类ID
- 如：`131` 或 `2469`

### 方法3：查看其他分类
如需要查询其他商品分类ID：
1. 使用抓包工具查看官方APP的请求
2. 找到 `productCategoryParentId` 参数
3. 记录对应的值

## 配置文件

配置保存在 `config.json` 中：
```json
{
    "category_id": "131",
    ...
}
```

## 验证方法

查看APP日志：
```
[REQUEST] GET .../getOrderPoolsList?productCategoryParentId=131
```

确保 `productCategoryParentId` 是正确的值。
